About VM and containers:

Before we have VM (its physical infrastructure inlude memory and process) -its not portable and scalable.different OS for each instances of VM
container run OS. OS can be shared between different containers. 
Run multiple containers in single OS with the help of docker abstraction
Container is single portable artifract /image to run the in a runnable environment. containers bundles all that app needs includes properties, libraies, App, dependncies.
containers is light weight and Vm is heavy weight

Docker and image:
Build container image /docker image. using docker engine , bring up the containers
In a container you you bundle everything that your application needs, so it is a single portable.build into an image is docker image.

Service bus:

Jboss mq for fuse, aks  service azure service bus.
transfer message from one queue to other 
queue partioning /topic partioning - load balancing 
handles multiple message in single jms session
DLQ- supports message that we dint  process (dead letter queue)
scheduled delivery - deliver message at specific time
batching - history of message at specific time
geo-diaster recovery - based on geogphical disaster , we can process other regions like ec/mw
it have integration with other azure services like event grip, logic app and fucntions
whenever new files comes in it creates new event and we can list to the event using logic app and function  
jboss - anycast address (one sender with multiple reciever , sender publishes message to it, only one recevier will get it  and multicast address sender publishes message to it, multiple reciever/each receriver will get copy of message
services bus, anycast replace by queues (we have virtual queues) and multi replace by topics (receiver listens to particular topic subscriptions)

jboss queues and topic creation: console, xml config, ansible playbook
aks queues and topc creation: azure portal , anisble playbook, ansible cli, arm template, powershel and rest api

create queues or topic in azure portal:
search the subscription FEI-AZ-Development-1
